# 🌐 ConnectForum – Community Discussion Platform

## 📌 Objective

The objective of this project is to design and develop a modern, interactive web-based forum platform where users can register, log in, create posts, and engage in discussions across various categories. The focus is on building a responsive UI, implementing client-side authentication, and managing user-generated content efficiently.

---

## ❗ Problem Statement

Traditional discussion platforms often lack modern UI/UX, personalization, and lightweight accessibility. Many existing systems are either too complex or not beginner-friendly.

This project aims to solve:

* Lack of simple and interactive discussion platforms
* Absence of visually appealing and responsive UI
* Limited understanding of frontend-driven application design
* Need for a lightweight forum system without heavy backend dependency

---

## 💡 Why This Project?

This project was developed to:

* Understand real-world web application structure
* Implement authentication and session handling
* Work with dynamic UI updates and state management
* Explore modern frontend technologies like Tailwind CSS
* Gain hands-on experience in building scalable UI systems

---

## 📊 Dataset Description

This project does not use an external dataset.

Instead, it uses:

* **LocalStorage** to store:

  * User data (name, email, hashed password)
  * Session data (logged-in user)
  * Posts created by users
  * Notifications

All data is stored and managed on the client side for demonstration purposes.

## ✨ Features

* User Registration & Login (with hashed passwords)
* Session Management (Remember Me functionality)
* Create and store posts
* Dynamic UI updates based on user state
* Notification system
* Dark/Light theme toggle
* Responsive design
* Modern card-based UI
* Category-based layout

---

## ⚠️ Limitations

* No backend (all data stored locally)
* Data is not persistent across devices
* No real-time updates
* Limited security (client-side authentication only)
* No comment/reply system implemented
* Posts are not fully dynamic in UI (basic rendering)
